# Telegram Bot with Local LLM

Этот проект представляет собой Telegram-бота, который использует локальную языковую модель (LLM) для генерации ответов. Бот запоминает историю диалога с каждым пользователем, поддерживает смену псевдонимов и автоматически обрезает контекст при превышении лимита токенов. Данные хранятся в MongoDB.

## Возможности

- Ответы на сообщения с помощью LLM (формат GGUF, библиотека `llama-cpp-python`).
- Сохранение истории диалога для каждого пользователя.
- Логирование всех запросов и ответов.
- Команды для управления:
  - `/help`, `/start` — справка.
  - `/name` — показать текущие псевдонимы пользователя и бота.
  - `/set_botname <имя>` — сменить имя бота.
  - `/set_username <имя>` — сменить имя пользователя.
  - `/tokens` — показать использованное количество токенов.
  - `/clear` — очистить историю диалога (оставить только стартовый промпт).
- Автоматическое создание профиля пользователя при первом обращении.

## Конфигурация

Параметры модели и путей задаются в файле `config.py`:

- `N_CTX` — размер контекста (по умолчанию 2048).
- `TEMPARETURE` — температура генерации (0.0–1.0, по умолчанию 0.85).
- `MAX_TOKENS` — максимальное количество токенов в ответе (по умолчанию 128).
- `MODEL_PATH` — путь к файлу модели в формате GGUF.
- `STARTER_PROMPT_PATH` — путь к текстовому файлу со стартовым промптом (должен содержать плейсхолдеры `<user alias>` и `<bot alias>`).
- `TOKEN` — токен Telegram-бота.
- `MONGODB_URI` — строка подключения к MongoDB.

## Примечания

- Бот автоматически обрезает историю, если количество токенов превышает `N_CTX - MAX_TOKENS`. Удаляются самые старые сообщения (кроме стартового промпта).
- При смене имени бота или пользователя все соответствующие префиксы в истории заменяются.
- Все взаимодействия логируются в поле `log`.
- Для работы с MongoDB используется асинхронный драйвер `motor`.
