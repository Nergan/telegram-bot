# Day Dating Bot

Телеграм-бот и встроенное WebApp для поиска единомышленников, гибкой фильтрации анкет и безопасного обмена контактами. Проект сочетает удобство текстового интерфейса мессенджера и интерактивность веб-интерфейсов для точной настройки предпочтений и управления запросами.

---

## Архитектура и ключевые возможности

*   **Пул профилей (Multi-profile):** Пользователь может создавать до 100 независимых анкет (личностей) в рамках одного аккаунта Telegram и мгновенно переключаться между ними, делая одну из них активной.
*   **Гибкий рендеринг медиакарточек:** Поддержка текстовых био, тегов и различных типов медиафайлов (фото, видео, анимации, голосовые сообщения, документы). Бот корректно обрабатывает как одиночные файлы, так и альбомы до 10 элементов.
*   **Разделение видимости контактов:**
    *   *Публичные контакты:* Отображаются непосредственно на карточке анкеты при просмотре другими пользователями.
    *   *Приватные контакты:* Скрыты по умолчанию и используются для обмена.
*   **Двухуровневая система обмена:**
    *   *Односторонний шеринг:* Прямая отправка своей карточки с выбранными приватными контактами и сопроводительным сообщением.
    *   *Взаимный обмен (Mutual Exchange):* Защищенный сценарий, при котором контактные данные обеих сторон раскрываются одновременно только после того, как получатель подтвердит запрос и выберет, какими из своих приватных контактов он готов поделиться взамен.
*   **Интерактивный WebApp-интерфейс (SPA):**
    *   *Управление тегами:* Визуальный выбор интересов для своей анкеты.
    *   *Продвинутый поиск:* Фильтрация анкет по логическим группам тегов: «Обязательно» (Must have), «Исключить» (Exclude) и «Любой из списка» (Any).
    *   *Центр запросов:* Единый хаб для мониторинга, принятия, отклонения или отмены активных предложений обмена.

## Структура репозитория

*   **`bot/`** — Клиентская логика Telegram-бота (на базе библиотеки **aiogram v3**):
    *   **`handlers/`** — Пакет модульных обработчиков, разделенный по зонам ответственности:
        *   `__init__.py` — Агрегация и приоритизация вложенных роутеров.
        *   `base.py` — Точка старта, отображение главного меню и глобальные FSM-отмены.
        *   `profile.py` — Управление пулом анкет, редактирование био и загрузка медиа.
        *   `contacts.py` — Добавление, удаление и настройка приватности контактов.
        *   `browse.py` — Механизм выдачи анкет и логика обработки запросов на обмен.
    *   `keyboards.py` — Шаблоны динамических Reply- и Inline-клавиатур.
    *   `helpers.py` — Вспомогательные функции для форматирования лимитов текста и генерации медиасообщений.
    *   `states.py` / `middlewares.py` — Машина состояний (FSM) и middleware для сквозного логирования действий.
*   **`core/`** — Системное окружение:
    *   `database.py` — Модуль асинхронного взаимодействия с MongoDB через драйвер **Motor**, включая историю просмотров (для предотвращения повторов в сессиях).
    *   `security.py` — Валидатор Telegram WebApp данных на основе HMAC-SHA256 с резервным механизмом авторизации.
    *   `config.py` — Глобальные переменные и база доступных тегов с синонимами для поиска.
*   **`webapp/`** — Веб-часть приложения (**FastAPI**):
    *   `api.py` — REST-API эндпоинты для синхронизации тегов, фильтров и обработки транзакций обмена контактами.
    *   **`static/index.html`** — Веб-интерфейс на чистом JavaScript и CSS, оптимизированный под встроенный контейнер Telegram WebApp.
*   **`main.py`** — Инициализация FastAPI приложения, управление жизненным циклом (lifespan), подключение базы данных, регистрация вебхуков и запуск фонового прослушивания обновлений Telegram.
