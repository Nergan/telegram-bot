# Telegram-бот знакомств и доски объявлений с поддержкой мультиаккаунтов

Асинхронный высокопроизводительный Telegram-бот с сопутствующим встроенным WebApp-интерфейсом, созданный на стеке **FastAPI**, **Aiogram 3** и **MongoDB**. Проект спроектирован в рамках чистой архитектуры (Clean Architecture / Domain-Driven Design), разделяя бизнес-логику и внешние фреймворки. Приложение работает как интерактивная доска объявлений, где пользователи находят собеседников по общим интересам, увлечениям или для свиданий с помощью гибкой системы анкет, тегов и настраиваемых фильтров поиска.

---

## Ключевые возможности

- **Мультипрофильность**: Пользователи могут создавать до 100 уникальных анкет (карточек) под разные цели (например, одна для поиска тиммейтов в играх, другая — для нетворкинга). При этом только одна выбранная анкета находится в статусе «Активна» и участвует в поиске.
- **Алгоритм продвинутого поиска**:
  * Полурандомный подбор кандидатов с учетом сложных фильтров тегов (`require_tags` — обязательные, `exclude_tags` — исключенные, `any_tags` — любые из списка).
  * Исключение пересечений: алгоритм исключает показ собственных анкет пользователя и тех анкет, которые он уже просмотрел.
  * Умный сброс истории: база автоматически очищает кэш просмотренных карточек для пользователя, если пул доступных кандидатов полностью исчерпан.
- **Механика обмена контактами**:
  * **Односторонняя отправка** (One-Way Share): Мгновенная отправка вашей активной анкеты и контактов (как публичных, так и выбранных приватных) получателю.
  * **Взаимный обмен контактами** (Mutual Exchange): Безопасный режим согласия. Ваши приватные контакты скрыты от получателя до тех пор, пока он не примет запрос и не выберет свои контакты для ответного обмена. Обмен происходит одновременно.
- **Интегрированное Telegram WebApp**:
  * Адаптивный HTML5-интерфейс для редактирования тегов анкеты, детальной настройки поисковых фильтров и управления разделом «Входящие/Исходящие запросы».
  * Безопасная валидация сессий с помощью криптографической проверки параметров `initData` (HMAC SHA256) для предотвращения подмены данных.
  * Возможность отправить профиль собеседника из WebApp прямо в личный чат с ботом для удобного продолжения диалога.
- **Группировка медиа и альбомов**: Бот корректно перехватывает альбомы (до 10 фото/видео в одном сообщении), временно буферизуя их перед сохранением в профиль, чтобы избежать дублирования триггеров состояний FSM.
- **Защита от спама и оптимизация логов**:
  * Middleware-фильтр ограничивает скорость нажатия кнопок клавиатуры и отправки текстовых сообщений.
  * Неблокирующий асинхронный логгер транзакций на базе внутренней очереди (`asyncio.Queue`). Запись логов пачками в MongoDB происходит раз в 2 секунды, исключая задержки на уровне СУБД при пиковых нагрузках.

## Структура репозитория (Чистая Архитектура)

Проект строго разграничивает зависимости между слоями:

```text
├── application/               # Слой приложения (Бизнес-сценарии / Юзкейсы)
│   └── services.py            # Сервисы взаимодействия со слоем инфраструктуры
├── bot/                       # Слой презентации Telegram-бота (Aiogram 3)
│   ├── handlers/              # Обработчики команд, сообщений и callback-запросов
│   │   ├── base.py            # Стартовые команды, главное меню, отмена состояний FSM
│   │   ├── browse.py          # Логика поиска, всплывающие окна, отправка запросов
│   │   ├── contacts.py        # Офлайн-валидатор контактов, настройка приватности
│   │   └── profile.py         # Управление анкетами и асинхронная сборка альбомов
│   ├── bot_setup.py           # Конфигурация инстанса бота и регистрация команд
│   ├── helpers.py             # Форматирование вывода карточек анкет и медиагрупп
│   ├── keyboards.py           # Шаблоны клавиатур Reply, Inline и WebApp кнопок
│   ├── middlewares.py         # Ограничение частоты запросов и локализация контекста
│   └── states.py              # Классы состояний конечного автомата (FSM)
├── domain/                    # Слой корпоративных правил (Domain Layer)
│   └── interfaces.py          # Интерфейсы репозиториев доступа к данным (Абстрактные классы)
├── infrastructure/            # Адаптеры инфраструктуры и фреймворков
│   ├── bot/
│   │   └── storage.py         # Кастомное хранилище состояний FSM на базе MongoDB
│   ├── config.py              # Инициализация переменных окружения
│   ├── database/
│   │   └── mongo_repository.py# Реализация репозиториев взаимодействия с Motor (MongoDB)
│   ├── locales.py             # Локализация на три языка (RU, EN, PT)
│   └── security.py            # Валидация подписи инициализирующих данных WebApp
├── webapp/                    # Слой презентации WebApp (FastAPI)
│   ├── api.py                 # REST API-эндпоинты (теги, фильтры, запросы, отправка карточек)
│   └── static/
│       └── index.html         # Одностраничное приложение (Vanilla JS + WebApp SDK)
├── main.py                    # Главная точка входа (FastAPI + Жизненный цикл Webhook)
└── .env                       # Конфиденциальные параметры окружения (скрыт от git)
```

## Используемый стек технологий

- **Фреймворки**: [FastAPI](https://fastapi.tiangolo.com/) (Web-интерфейс, API-эндпоинты), [Aiogram 3.x](https://docs.aiogram.dev/) (Асинхронный фреймворк для ботов)
- **База данных**: [MongoDB](https://www.mongodb.com/) с асинхронным клиентом [Motor](https://motor.readthedocs.io/)
- **Валидация данных**: [Pydantic v2](https://docs.pydantic.dev/)
- **Фронтенд**: Веб-интерфейс на чистом JS с использованием официального Telegram WebApp SDK
- **Утилиты**: `phonenumbers` (валидация номеров в международном формате)

## Инструкция по установке и запуску

### 1. Системные требования
- Установленный Python версии 3.10 или выше.
- Доступ к активной СУБД MongoDB (локальной или облачной Atlas).
- Токен Telegram-бота, полученный у [@BotFather](https://t.me/BotFather).
- Программа туннелирования для локальной разработки с поддержкой SSL (например, [ngrok](https://ngrok.com/)).

### 2. Клонирование и настройка окружения
Клонируйте репозиторий на локальную машину и установите зависимости:
```bash
git clone https://github.com/yourusername/day_dating_bot.git
cd day_dating_bot

python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```

*(Примечание: Убедитесь, что ваш файл `requirements.txt` содержит библиотеки `fastapi`, `uvicorn`, `aiogram`, `motor`, `pydantic`, `phonenumbers`, `python-dotenv`.)*

### 3. Настройка конфигурации
Создайте файл `.env` в корневом каталоге проекта:
```env
TOKEN="ВАШ_ТОКЕН_TELEGRAM_БОТА"
MONGODB_URI="mongodb+srv://user:pass@cluster.mongodb.net/database?retryWrites=true&w=majority"
WEBHOOK_URL="https://ваш-поддомен-ngrok.ngrok-free.app"
WEBHOOK_SECRET="dating_bot_secure_path_abc123"
```

### 4. Запуск приложения
Запустите веб-сервер FastAPI с помощью Uvicorn. На этапе инициализации обработчик `lifespan` автоматически настроит необходимые коллекции и индексы в MongoDB, применит миграции к старым профилям, запустит фоновый процесс логирования и зарегистрирует webhook в Telegram.

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

## Детали поискового алгоритма

При вызове функции **Поиск**:
1. Приложение считывает поисковые фильтры из активного профиля пользователя (обязательные, исключенные и пересекающиеся теги).
2. Генерируется случайное число с плавающей точкой `random_index`, и база данных пытается найти один документ со значением `random_index >= сгенерированного_числа`.
3. Если совпадений в прямом направлении не найдено, запускается поиск в обратном направлении (`random_index < сгенерированного_числа`).
4. Если результаты по-прежнему отсутствуют, алгоритм игнорирует список просмотренных анкет. Если анкета находится при таком условии, кэш просмотренных карточек сбрасывается, запуская мягкий повторный цикл показа анкет.

## Рекомендации по развертыванию в продакшене

Для обеспечения отказоустойчивости и безопасности рекомендуется разворачивать проект за обратным прокси-сервером (например, **Nginx**) с настроенным SSL-сертификатом.

### Пример конфигурации Nginx:
```nginx
server {
    listen 443 ssl http2;
    server_name bot.example.com;

    ssl_certificate /etc/letsencrypt/live/bot.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/bot.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded-for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Для непрерывной работы процесса в системе используйте диспетчеры процессов (например, **systemd** или **PM2**).
